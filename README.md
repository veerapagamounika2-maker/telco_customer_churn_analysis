# Telecom Customer Churn Analysis

# Project Overview
This project analyzes and predicts customer churn in a telecom company using SQL, Machine Learning, and Power BI.  
The goal is to identify high-risk customers and provide actionable business insights.

---

#  Dataset
Kaggle Telco Customer Churn Dataset  
Total Records: ~7,000 customers

---

#Tools & Technologies
- SQL Server – Data Cleaning & Analysis  
- Python (Pandas, Scikit-learn) – Machine Learning  
- Google Colab – Model Development  
- Power BI – Dashboard & Visualization  
- GitHub – Version Control  

---

#Project Workflow
1. Imported and cleaned data using SQL  
2. Performed Exploratory Data Analysis  
3. Created new features for modeling  
4. Built ML classification models  
5. Designed interactive Power BI dashboard  
6. Generated business insights  

---

#SQL Analysis
- Data validation and cleaning  
- Customer segmentation  
- Churn rate calculation  
- Contract-wise and gender-wise analysis  

File: `churn_analysis.sql`

---

# Machine Learning Model
- Models Used: Logistic Regression, Random Forest  
- Evaluation Metric: Accuracy  

Model Accuracy: **85%**

Notebook: `churn.ipynb`

---

# Key Insights
- Month-to-month customers have the highest churn rate  
- New customers churn more than long-term users  
- High monthly charges increase churn risk  
- Electronic check users show higher churn
